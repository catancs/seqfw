# Benchmark corpus provenance & ethics

Every `known-bad/` file is a **self-built structural reproducer** for an
already-fixed, publicly-disclosed bug class. None contains shellcode or a live
exploit payload; the hazardous property is purely structural — a count or length
field that lies about its payload, or an over-long FORMAT row — which is exactly
what `seqfw` rejects at the trust boundary. This satisfies the design spec's
disclosure policy (§11): **patch-gated, generators over weaponized files,
provenance per sample, nothing shipped for an unfixed issue.**

| file | bug class | reference | upstream fix |
|---|---|---|---|
| `vcf_format_overflow.vcf` | VCF FORMAT field-count overflow → OOB write in `vcf_parse_format` | CVE-2020-36403 | htslib ≤1.10.2, fixed 1.11 |
| `gzi_impossible_count.gzi` | BGZF/GZI index integer overflow (impossible entry count) | CVE-2026-31970 class | htslib, fixed 1.21.1/1.22.2/1.23.1 |
| `csi_impossible_nref.csi` | CSI index implausible `n_ref` count | CVE-2026-31970 class | htslib, as above |
| `gzip_bomb.fastq.gz` | decompression bomb (high expansion ratio) | resource-exhaustion class (no CVE) | n/a — defense, not a parser bug |
| `fastq_truncated.fastq` | truncated FASTQ record (corruption) | corruption/truncation class | n/a |
| `fasta_dupnames.fasta` | duplicate FASTA sequence name (corruption) | corruption class | n/a |

The files are regenerated by `gen_corpus.py`; only `corpus/manifest.json`
(SHA-256 + provenance per file) is version-controlled, not the generated bytes.
