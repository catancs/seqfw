# seqfw security benchmark

A reproducible harness that produces three honest numbers (design spec §8):

1. **Block rate** — % of a known-bad set that `seqfw` rejects at the gate.
2. **Harm prevented** — inputs that crash a *real, pinned, unmodified,
   ASAN+UBSan-built* vulnerable tool (htslib 1.10.2) but that `seqfw` rejects
   first.
3. **False positives + overhead** — over a known-good set (must reject ~0%),
   plus per-file latency.

## Run it

```bash
make benchmark-local   # numbers 1 + 3, deterministic, no Docker required
make benchmark         # adds number 2 — builds pinned htslib under ASAN in Docker
```

`benchmark-local` shells out to the release `seqfw` binary against the generated
corpus and exits non-zero on <100% block rate or any false positive, so it also
works as a regression gate.

## Corpus & ethics

`gen_corpus.py` builds the corpus; only `corpus/manifest.json` is committed (the
generated bytes are reproducible from the script). Every known-bad file is a
patch-gated, self-built **structural** reproducer — see `PROVENANCE.md`. No live
crasher is shipped for any unfixed issue.
